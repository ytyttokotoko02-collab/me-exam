ME技術実力検定試験 問題集システム 運用マニュアル
---
このシステムについて
第２種ME技術実力検定試験の過去問を、ブラウザ上で解くことができるWebシステムです。  
回ごと・午前午後ごとに独立したURLでアクセスできます。
アクセスURLの例
```
https://【ユーザー名】.github.io/me-exam/round46_am/   ← 第46回 午前
https://【ユーザー名】.github.io/me-exam/round46_pm/   ← 第46回 午後
https://【ユーザー名】.github.io/me-exam/round47_am/   ← 第47回 午前
https://【ユーザー名】.github.io/me-exam/round47_pm/   ← 第47回 午後
```
---
フォルダ・ファイル構成
```
me-exam/                        ← GitHubリポジトリのルート
│
├── checker.html                ← データ検証ツール（共通・変更不要）
├── README.md                   ← このファイル
│
├── round46_am/                 ← 第46回 午前
│   ├── index.html              ← 問題ページ（基本的に変更不要）
│   ├── round46_am.json         ← 問題データ
│   └── images/                 ← 図の画像ファイル置き場
│       ├── 46_am_q12.png
│       └── ...
│
├── round46_pm/                 ← 第46回 午後（追加予定）
│   ├── index.html
│   ├── round46_pm.json
│   └── images/
│
└── round47_am/                 ← 第47回 午前（追加予定）
    ├── index.html
    ├── round47_am.json
    └── images/
```
各ファイルの役割
ファイル	役割	編集頻度
`index.html`	問題を解く画面。JSONを読み込んで動作する	基本不要
`checker.html`	JSONの整合性チェック・目視確認ツール	不要
`round○○_○○.json`	問題・正解データ本体	回ごとに新規作成
`images/`	図が含まれる問題の画像ファイル置き場	画像追加時のみ

新しい回・セッションを追加する手順
ステップ1：既存フォルダをまるごとコピーする
`round46_am` フォルダをコピーして、新しい名前に変更します。
```
フォルダ名のルール： round【回次】_【am=午前 / pm=午後】

例）
第46回 午後 → round46_pm
第47回 午前 → round47_am
第47回 午後 → round47_pm
```
フォルダの中身はそのままでOKです。次のステップでJSONだけ差し替えます。
ステップ2：JSONファイルを作成・編集する
コピーしたフォルダ内のJSONファイルを新しい名前に変更し、テキストエディタで内容を編集します。
① ファイル名を変更する
```
例）round46_pm フォルダの場合
変更前: round46_am.json
変更後: round46_pm.json
```
② examブロックを編集する（ファイル先頭部分）
```json
"exam": {
  "round": 46,        ← 回次を数字で入力
  "session": "午後",  ← "午前" または "午後"
  "total": 60         ← 総問題数（通常は60）
},
```
③ questionsブロックに問題データを入力する
問題1つ分のデータ構造：
```json
{
  "no": 1,
  "text": "問題文をここに入力する。",
  "image": null,
  "choices": [
    "選択肢1",
    "選択肢2",
    "選択肢3",
    "選択肢4",
    "選択肢5"
  ],
  "answer": 5
}
```
項目	説明
`no`	問題番号（1〜60の整数）
`text`	問題文（文字列）
`image`	図がない場合は `null`、ある場合はファイル名（例：`"46_pm_q5.png"`）
`choices`	選択肢を配列で5つ記入
`answer`	正解の選択肢番号（1〜5の整数）。PDFの番号をそのまま入力する
> ⚠️ `answer` は **元のPDFの選択肢番号** をそのまま入力してください。  
> システムが自動でシャッフルに対応します。
④ index.htmlのJSON読み込み先を変更する
コピーしたフォルダ内の `index.html` をテキストエディタで開き、先頭付近の1行を編集します。
```javascript
// 例）round46_pm フォルダの場合
// 変更前
const JSON_FILE = 'round46_am.json';

// 変更後
const JSON_FILE = 'round46_pm.json';
```
ステップ3：checker.htmlで整合性を確認する
`checker.html` をブラウザで開き、作成したJSONファイルをドラッグ＆ドロップします。
以下の項目が すべて緑（✓） になっていればOKです。
回次・セッション・総問題数の定義あり
問題数が定義値と一致している
問番号に重複・抜けがない
全問題の選択肢数が5択
全問題の正解番号が1〜5の範囲内
全問題の問題文あり
エラーが出た場合は、該当の問題番号が表示されるので修正してください。
また、ページ下部の 問題一覧テーブル で、PDFと正解番号・正解テキストを1問ずつ目視確認することをお勧めします。
ステップ4：GitHubにアップロードする
新しく作成したフォルダごとGitHubにアップロードします（詳細は後述）。  
アップロード完了後、以下のURLでアクセスできます。
```
https://【ユーザー名】.github.io/【リポジトリ名】/round46_pm/
```

図（画像）を追加する手順
ステップ1：画像ファイルを用意する
PDFを開き、図が含まれる問題の図をスクリーンショットで切り取ります。
ファイル名のルール：`【回次】_【am/pm】_q【問番号】.png`
```
例）
第46回午前 問12の図 → 46_am_q12.png
第46回午後 問5の図  → 46_pm_q5.png
```
ステップ2：imagesフォルダに保存する
画像ファイルを該当フォルダの `images/` 内に入れます。
```
例）第46回午前 問12の図の場合
round46_am/images/46_am_q12.png
```
ステップ3：JSONのimageフィールドを更新する
対象問題の `"image": null` をファイル名に書き換えます。
```json
// 変更前
{ "no": 12, "text": "図の波形Aを積分した波形はどれか。", "image": null, ... }

// 変更後
{ "no": 12, "text": "図の波形Aを積分した波形はどれか。", "image": "46_am_q12.png", ... }
```

GitHub Pagesへの公開手順
初回のみ：リポジトリ作成とPages設定
GitHub にサインインする
右上の「＋」→「New repository」をクリック
Repository name に任意の名前を入力（例：`me-exam`）
「Public」を選択して「Create repository」をクリック
「uploading an existing file」をクリックし、以下をアップロードする
`checker.html`
`README.md`
`round46_am/` フォルダ内のファイルをすべて（フォルダ構造を保って）
「Commit changes」をクリック
リポジトリの「Settings」→「Pages」を開く
Source を「Deploy from a branch」、Branch を「main」→「/(root)」に設定して「Save」
数分待つと以下のURLで公開される
```
https://【ユーザー名】.github.io/me-exam/round46_am/
```
新しい回を追加するとき
GitHubのリポジトリページを開く
「Add file」→「Upload files」をクリック
新しいフォルダ（例：`round46_pm/`）内のファイルをすべてアップロード
※ GitHubのWeb画面ではフォルダごとドラッグ＆ドロップできます
「Commit changes」をクリック
数分後に新しいURLでアクセスできるようになる
JSONや画像を更新するとき
リポジトリで対象ファイルをクリック
右上の鉛筆アイコン（Edit）で直接編集するか、「Upload files」で上書きアップロード
「Commit changes」をクリックすると自動で反映される

トラブルシューティング
「データの読み込みに失敗しました」と表示される
HTMLファイルをローカル（自分のPC）で直接開いていませんか？  
このシステムはWebサーバー上（GitHub Pages等）でのみ動作します。
`index.html` 内の `JSON_FILE` の値と、実際のJSONファイル名が一致しているか確認してください。
JSONファイルが `index.html` と同じフォルダに置かれているか確認してください。
画像が表示されず「画像ファイルが見つかりません」と表示される
`images/` フォルダに画像ファイルが正しくアップロードされているか確認してください。
JSONの `image` フィールドのファイル名と、実際のファイル名（大文字・小文字を含む）が完全に一致しているか確認してください。
checker.htmlでエラーが出る
選択肢数エラー：該当問題の `choices` 配列の要素数が5つになっているか確認してください。
正解番号エラー：`answer` が1〜5の整数になっているか確認してください。`"5"` のように文字列になっていないか注意してください。
問番号の重複・抜け：問題をコピー&ペーストした際に `no` の値が重複していないか確認してください。
